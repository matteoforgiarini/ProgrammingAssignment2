machine learning: random forest

In this project we have to perform a random forest analysis to estimate ÒClasseÓ of  exercises performed by 6 people.

1 Classe is correct, 4 are wrong.

Data provided contain lots of data from many body sensors. Data from sensors are the features of whatever model we can build.

I tried a random forest analysis to predict 5 different classes.

Original data contain 160 columns. Many of them are almost empty or containing bad data. This is statistical noise, you need to find it and clean it so that the model can get just the signal of the data to predict the outcome.

dropping columns with more than 50% NaN

d1=d[, colSums(is.na(d)) < nrow(d) * 0.5]  

 

Dropping columns with bad data (div/0). I also dropped columns with data about ID subject, time, day and number of exercise. I do not think this information can be useful to predict the outcome. Either they  lead to over fitting  if they are correlated with the outcome, or they lower the accuracy. Broadly speaking, from my point of view, this is noise, not signal.

d2=d1[-c(1:5)]
d2=d2[-c(7:15)]
d2=d2[-c(1:2,29:34, 38:46, 60:68)]

I ended up with 53 variables in my dataset.

dim(d2)

## [1] 19622    53

This dataset contains lots of data, with many repetition of each subject. I tried  different trials on my I5, 2 cores Mac air with 8GB RAM. Using 1/7 of the dataset allows me to get good performances during data analysis without lowering model accuracy.

d3=d2[(sample(1:nrow(d1), size = nrow(d1)/7, replace = F)),]
dim(d3)

## [1] 2803   53

feat=d3[-c(53)]

I used Cross Validation to find the right number of feature for this model.

I used a 5-folds cross validation to balance predictiveness and performance on my Mac.

Results show a very nice pattern: 13 features are able to predict the outcome with almost the same accuracy as 52 variables.

cv=rfcv(feat, d3$classe, cv.fold=5)

I used VarImp function to get the importance of the variable in the model

imp=(v$importance)
imp$var=row.names(imp)
impor=imp[with(imp, order(-imp$Overall)), ]

Random Forest predicts the outcome with a out of sample error (1-0.93)=6.7%

Random forest automatically perform cross validation by sampling trees and by averaging the prediction error on the sample non included in the samples used for training the model.

This model with 13 variable allowed me to get 100% accuracy prediction with the test dataset provided for the second part of the project.
